Прочитай файл `PLAN.md`, посмотри проект, покрой код нужными тестами и проведи тестирование.

### Требования:

1. Понимание требований из PLAN.md
2. Создание тестов для нового функционала
3. Наличие unit тестов для функционала
4. Наличие e2e тестов для функционала

### Формат вывода:

Создай файл `TEST_REPORT.md` с описанием:

- Какие тесты изменены/созданы
- Какие тесты проведены
- Отчет о запуске тестов
